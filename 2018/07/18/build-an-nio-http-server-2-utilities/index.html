DynamicByteBuffer

Although we have the ByteBuffer in the java.nio package, ByteBuffer is a fixed length buffer. In fact, we could not know in advance the length of the message that sent by the client. On the other hand, we cannot use ByteArrayOutputStream either, because it cannot be accessed at arbitrary index. Thus we have to implement a DynamicByteBuffer whose capacity can be expand on the fly and which we could access value at any index at any time.

First, let us draw the skeleton of our DynamicByteBuffer:

As you may have seen, we are going to implement five main methods, that is, get and its overloaded version, pop, append and toByteArray.
We store the bytes inside the ‘block’s of blocks with each block stores exactly BLOCK_BYTE_COUNT bytes and the blocks are placed in order of the data bytes. Furthermore, the blocks could be expanded at runtime if there is no enough space to store extra bytes. And that is why we store bytes in blocks instead of storing them in a array directly like java.util.ArrayList (or more specifically it’s more like vector<T> of C++ STL). By the aid of the using of blocks, we could save a lot of time onarraycopy when expanding.
beginCursor and endCursor are a pair of cursors that shows the limit of our buffer, anything outside the limits should not be considered as valid contents.
